# Search Console Context

## Papel no agente

O Google Search Console representa a demanda orgânica real do site.

Enquanto o Google Ads mostra performance paga, custo e conversão, o Search Console mostra:
- quais termos já geram impressões orgânicas;
- quais termos geram cliques orgânicos;
- em quais termos o site aparece, mas com posição ruim;
- onde há oportunidade de SEO, mídia paga ou ambos.

## Métricas principais

- query: termo pesquisado no Google;
- clicks: cliques orgânicos;
- impressions: impressões orgânicas;
- ctr: taxa de clique orgânica;
- position: posição média orgânica.

## Interpretação base

### Alta impressão + posição ruim
Indica demanda existente, mas baixa captura orgânica.

Ações possíveis:
- criar keyword exata no Google Ads;
- criar campanha/grupo específico;
- revisar landing page;
- criar conteúdo SEO;
- ajustar copy dos anúncios.

### Alta impressão + CTR baixo
Indica que o site aparece, mas o snippet/oferta não atrai clique.

Ações possíveis:
- revisar title/meta description;
- melhorar copy dos anúncios pagos;
- testar proposta de valor mais direta;
- verificar se a intenção da busca bate com a landing.

### Posição boa + CTR alto
Indica força orgânica.

Ações possíveis:
- manter monitoramento;
- evitar escalar mídia paga agressivamente sem necessidade;
- proteger termos branded;
- avaliar economia de mídia em termos não estratégicos.

### Baixa impressão + alto custo no Ads
Indica possível desalinhamento entre mídia paga e demanda orgânica.

Ações possíveis:
- revisar intenção;
- verificar se o termo é muito nichado;
- avaliar negativação;
- avaliar se é termo de descoberta ou teste controlado.

## Cuidados

- Não cortar automaticamente termos apenas porque o orgânico é forte.
- Termos branded devem ser protegidos mesmo com boa posição orgânica.
- Search Console não informa conversão diretamente.
- Decisões finais devem cruzar Search Console com Google Ads.
